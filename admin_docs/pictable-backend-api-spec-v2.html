찰칵밥상 v2

백엔드 API 명세

프론트 연동 기준 최신 API와 Django Admin CRUD 범위

API 설계 기준

  • Base URL은 https://pictable.online/api 이며, Vercel Rewrite를 통해 EC2 Django 서버로 전달된다.
  • 인증은 JWT Access/Refresh token과 Google OAuth code exchange를 사용한다.
  • /api, /media, /admin 요청은 프론트 도메인에서 백엔드로 프록시된다.

인증 / 회원

MethodEndpoint역할
POST/api/auth/signup/이메일 회원가입
POST/api/auth/login/이메일 로그인 및 JWT 발급
POST/api/auth/google/Google OAuth 인증 코드 교환
POST/api/auth/refresh/Access token 재발급
POST/api/auth/logout/Refresh token 블랙리스트 처리

사용자 / 마이페이지

MethodEndpoint역할
GET/api/users/me/내 프로필 조회
PATCH/api/users/me/내 프로필 수정
GET/api/users/me/saved-recipes/저장한 레시피 조회
GET/api/users/me/liked-recipes/좋아요한 레시피 조회
GET/api/users/me/recipes/내가 작성한 레시피 조회
POST/api/users/{userId}/subscribe/사용자 구독 / 구독 취소

레시피 / 추천

MethodEndpoint역할
GET/api/recipes/공개 레시피 목록 조회
POST/api/recipes/레시피 생성
GET/api/recipes/{recipeId}/레시피 상세 조회
PATCH/api/recipes/{recipeId}/레시피 수정
DELETE/api/recipes/{recipeId}/레시피 삭제
GET/api/recipes/ingredients/?search=계란재료 자동완성
GET/api/recipes/recommendations/?ingredients=계란,김치,밥선택 재료 기반 추천

피드 / 소셜

MethodEndpoint역할
GET/api/feeds/?sort=popular&search=김치&tag=한식&cursor=...피드 목록 조회
GET/api/feeds/tags/피드 태그 조회
POST/api/recipes/{recipeId}/like/좋아요 토글
POST/api/recipes/{recipeId}/save/저장 토글
GET/api/recipes/{recipeId}/comments/댓글 목록 조회
POST/api/recipes/{recipeId}/comments/댓글 작성

미디어 / AI 재료 감지

MethodEndpoint역할
POST/api/media/upload/이미지 업로드 및 재료 감지 작업 생성
GET/api/media/{mediaId}/업로드 미디어 조회
GET/api/media/detection/{jobId}/재료 감지 작업 결과 조회

Django Admin CRUD 범위

관리자 화면에서 사용자, OAuth 계정, 구독 관계, 레시피, 태그, 재료, 단계, 이미지, 좋아요, 저장, 댓글, 미디어, 재료 감지 작업/결과, 알림 모델을 관리한다. 레시피는 재료/단계/이미지를 inline으로 함께 편집할 수 있다.
